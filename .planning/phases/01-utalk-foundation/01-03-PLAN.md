<objective>
**Create the UTalk API client service for authenticating with and consuming the UTalk (Umbler Talk) WhatsApp API.**

**Purpose:** The UTalk client provides module-level async functions for all UTalk API operations — authentication, channel listing, chat management, message send/receive, webhook management, and connection testing. It follows the exact pattern established by `evolution-api.ts`: fetch-based HTTP, Bearer token auth, per-endpoint named functions, and graceful error handling with `{ ok: false, error }` returns. Encrypted credentials stored in `utalk_integrations` are decrypted at runtime using `server/lib/encryption.ts`.

**Output:** `server/services/utalk-client.ts` (API client service) and an update to `.env.example` documenting the new `UTALK_API_URL` variable.
</objective>

  <action>
    Create `server/services/utalk-client.ts` following the exact `evolution-api.ts` pattern.

    **File structure:**

    1. `// @ts-nocheck` at the top

    2. Imports:
    ```typescript
    import { getSupabase } from '../db/supabase.js';
    import { decrypt } from '../lib/encryption.js';
    ```

    3. Module-level config constant:
    ```typescript
    const UTALK_API_URL = process.env.UTALK_API_URL || 'https://app-utalk.umbler.com/api';
    ```

    4. **getDecryptedToken(tenantId: string): Promise<string | null>**
       - Queries `utalk_integrations` table for the tenant's record
       - If found, decrypts `api_key_encrypted` using `decrypt()` from encryption.ts
       - Returns the plaintext token or null
       - Logs with emoji prefix: `🔑 UTalk token decrypted for tenant`

    5. **getOrganizationId(apiToken: string): Promise<string | null>**
       - `GET ${UTALK_API_URL}/v1/members/me/`
       - Headers: `Authorization: Bearer ${apiToken}`
       - Parses response body, extracts `organizationId` from the response
       - Returns the org ID string or null
       - Console.log: `🔌 UTalk auth: orgId=${orgId}`

    6. **listChannels(apiToken: string, organizationId: string): Promise<any[]>**
       - `GET ${UTALK_API_URL}/v1/channels/?organizationId=${organizationId}`
       - Returns array of channel objects or empty array

    7. **listChats(apiToken: string, organizationId: string): Promise<any[]>**
       - `GET ${UTALK_API_URL}/v1/chats/?organizationId=${organizationId}`
       - Returns array of chat objects or empty array

    8. **getMessages(apiToken: string, chatId: string, startDate?: string): Promise<any[]>**
       - `GET ${UTALK_API_URL}/v1/chats/${chatId}/relative-messages/`
       - If startDate provided, append as query param
       - Returns array of message objects or empty array

    9. **sendSimplifiedMessage(apiToken: string, toPhone: string, fromPhone: string, organizationId: string, message: string): Promise<any>**
       - `POST ${UTALK_API_URL}/v1/messages/simplified/`
       - Body: `{ number: toPhone.replace(/\D/g, ''), text: message, user: fromPhone, organizationId }`
       - Follows the evolution-api.ts pattern: number cleaning with `.replace(/\D/g, '')`
       - Console.log: `📨 UTalk sendSimplifiedMessage to=${toPhone}`

    10. **sendMessage(apiToken: string, chatId: string, message: string): Promise<any>**
        - `POST ${UTALK_API_URL}/v1/messages/`
        - Body: `{ chatId, text: message }`
        - Used for sending to an existing chat/contact pair

    11. **listWebhooks(apiToken: string, organizationId: string): Promise<any[]>**
        - `GET ${UTALK_API_URL}/v1/webhooks/?organizationId=${organizationId}`
        - Returns array of webhook config objects or empty array

    12. **listContacts(apiToken: string, organizationId: string): Promise<any[]>**
        - `GET ${UTALK_API_URL}/v1/contacts/?organizationId=${organizationId}`
        - Returns array of contact objects or empty array

    13. **testConnection(apiToken: string): Promise<{ ok: boolean; error?: string; organizationId?: string }>**
        - Calls `getOrganizationId(apiToken)` as a connectivity test
        - If orgId returned: `return { ok: true, organizationId: orgId }`
        - If error: `return { ok: false, error: e.message }`
        - Console.log: `✅ UTalk connection OK` or `❌ UTalk connection failed: ${error}`
        - This is the primary endpoint for the Phase 2 "test connection" feature

    **Guard pattern (every exported function except getDecryptedToken):**
    ```typescript
    if (!apiToken) {
      return { ok: false, error: 'UTalk API token not provided' };
    }
    ```

    **Error handling pattern (every exported function):**
    ```typescript
    try {
      const res = await fetch(...);
      return await res.json();
    } catch (e: any) {
      console.error('UTalk functionName error:', e.message);
      return { ok: false, error: e.message };
    }
    ```

    **Critical patterns to follow (matching evolution-api.ts):**
    - Use `fetch()` for HTTP, never axios
    - Use `Authorization: Bearer` header pattern (not `apikey` header like Evolution — UTalk uses Bearer token)
    - Use emoji console.log prefixes: 🔌 UTalk, 📨 UTalk, ✅ UTalk, ❌ UTalk, ⚠️ UTalk
    - Return `{ ok: false, error: '...' }` on failure, matching the existing evolution-api.ts convention
    - Number cleaning: `.replace(/\D/g, '')` same as evolution-api.ts line 59
    - Early-return guard with warning message when config is missing
    - All functions are module-level named exports (no class)
  </action>
